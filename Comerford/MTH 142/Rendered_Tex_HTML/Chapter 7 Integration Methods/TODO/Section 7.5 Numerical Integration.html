Under- or Over-estimate?

Something we already saw in Math 141:
  • If f f fff is increasing on [ a , b ] [ a , b ] [a,b][a, b][a,b], then
LEFT ( n ) ≤ ∫ a b f ( x ) d x ≤ RIGHT ( n ) LEFT ( n ) ≤ ∫ a b   f ( x ) d x ≤ RIGHT ( n ) LEFT(n) <= int_(a)^(b)f(x)dx <= RIGHT(n)\operatorname{LEFT}(n) \leq \int_{a}^{b} f(x) d x \leq \operatorname{RIGHT}(n)LEFT(n)≤∫abf(x)dx≤RIGHT(n)
  • If f f fff is decreasing on [ a , b ] [ a , b ] [a,b][a, b][a,b], then
RIGHT ( n ) ≤ ∫ a b f ( x ) d x ≤ LEFT ( n ) RIGHT ( n ) ≤ ∫ a b   f ( x ) d x ≤ LEFT ( n ) RIGHT(n) <= int_(a)^(b)f(x)dx <= LEFT(n)\operatorname{RIGHT}(n) \leq \int_{a}^{b} f(x) d x \leq \operatorname{LEFT}(n)RIGHT(n)≤∫abf(x)dx≤LEFT(n)

For the midpoint and trapezoid rules, whether we get an under- or over-estimate is determined not by whether f f fff is increasing/decreasing, but whether f f fff is concave up or concave down.
For the trapezoid rule:

  • If f f fff is concave down, the trapezoid rule underestimates.
  • If f f fff is concave up, the trapezoid rule overestimates.
For the midpoint rule:

  • If f f fff is concave down, the midpoint rule overestimates.
  • If f f fff is concave up, the midpoint rule underestimates.

What we have found

  • If f f fff is concave up on [ a , b ] [ a , b ] [a,b][a, b][a,b], then
MID ( n ) ≤ ∫ a b f ( x ) d x ≤ TRAP ( n ) MID ( n ) ≤ ∫ a b   f ( x ) d x ≤ TRAP ( n ) MID(n) <= int_(a)^(b)f(x)dx <= TRAP(n)\operatorname{MID}(n) \leq \int_{a}^{b} f(x) d x \leq \operatorname{TRAP}(n)MID(n)≤∫abf(x)dx≤TRAP(n)

Some final remarks

  • LEFT ( n ) LEFT ( n ) LEFT(n)\operatorname{LEFT}(n)LEFT(n) and RIGHT ( n ) ( n ) (n)(n)(n) can be thought of as attempts to approximate the graph of f f fff using horizontal and vertical lines (like a computer screen). The error behaves like 1 n 1 n (1)/(n)\frac{1}{n}1n, and the more f f fff decreases or increases (i.e. the bigger f ′ f ′ f^(')f^{\prime}f′ is), the worse the approximation will be.
  • MID ( n ) MID ( n ) MID(n)\operatorname{MID}(n)MID(n) and TRAP ( n ) TRAP ( n ) TRAP(n)\operatorname{TRAP}(n)TRAP(n) can be thought of as attempts to approximate the graph of f f fff using (sloping) straight lines. The error behaves like 1 n 2 1 n 2 (1)/(n^(2))\frac{1}{n^{2}}1n2, and the more the graph of f f fff bends (i.e. the bigger f ′ ′ f ′ ′ f^('')f^{\prime \prime}f′′ is), the worse the approximation will be.
  • SIMP ( n ) SIMP ( n ) SIMP(n)\operatorname{SIMP}(n)SIMP(n) can be thought of as an attempt to approximate the graph of f f fff using segments of parabolas. The error behaves like 1 n 4 1 n 4 (1)/(n^(4))\frac{1}{n^{4}}1n4, and the bigger the fourth derivative f ( i v ) f ( i v ) f^((iv))f^{(i v)}f(iv) is, the worse the approximation will be.
The better the desired approximation, the better it is to use Simpson's rule.
We can also use higher degree polynomials than quadratics as in Simpson. This is called quadrature.
